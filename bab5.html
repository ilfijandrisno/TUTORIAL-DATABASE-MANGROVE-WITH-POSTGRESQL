Bab 5: Memfilter Data dengan WHERE

Pada bab ini, kita akan mempelajari cara memfilter data konservasi mangrove berdasarkan kondisi tertentu menggunakan klausa WHERE, penting untuk analisis terfokus dalam sistem blockchain.

Deskripsi Kasus

Organisasi ingin memfilter proyek dengan luas area lebih dari 50 hektar untuk fokus pada proyek besar yang memiliki dampak signifikan dalam transaksi karbon blockchain.

Langkah untuk Pemula

1. Filter Dasar

SELECT * FROM mangrove_conservation_records WHERE area_ha > 50;

Penjelasan Query:

  • SELECT * - Memilih semua kolom dari tabel
  • FROM mangrove_conservation_records - Menentukan tabel sumber data
  • WHERE area_ha > 50 - Kondisi filter untuk menampilkan hanya proyek dengan luas area > 50 hektar

Output Contoh:

conservation_id location area_ha carbon_credits
C002 Takalar 75 370
C004 Lampung Barat 60 300

Analisis: Query ini membantu mengidentifikasi proyek-proyek besar yang memiliki dampak lingkungan lebih signifikan dan potensi kredit karbon lebih tinggi.

2. Kondisi Ganda

SELECT * FROM mangrove_conservation_records WHERE area_ha > 50 AND carbon_credits > 300;

Penjelasan Query:

  • AND - Operator logika untuk menggabungkan dua kondisi
  • Hanya menampilkan baris yang memenuhi kedua kriteria: luas >50ha DAN kredit >300

Output Contoh:

conservation_id location area_ha carbon_credits
C002 Takalar 75 370
C006 Labuhan Batu 80 400

Analisis: Hasil ini sangat berguna untuk menentukan proyek prioritas dalam sistem blockchain yang memerlukan dampak ganda (luas area dan kredit karbon).

Untuk Pengguna Menengah

Gunakan IN dan BETWEEN

SELECT * FROM mangrove_conservation_records WHERE location IN ('Aceh Jaya', 'Takalar') AND date_recorded BETWEEN '2024-01-01' AND '2024-12-31';

Penjelasan Query:

  • IN - Memfilter lokasi yang termasuk dalam daftar tertentu
  • BETWEEN - Memfilter tanggal dalam rentang tertentu
  • Gabungan kondisi untuk analisis regional dan temporal

Kegunaan: Query ini membantu analisis proyek di wilayah tertentu dalam periode waktu tertentu untuk pelaporan blockchain triwulanan.

Untuk Pengguna Tingkat Lanjut

Gunakan LIKE untuk Pencocokan Pola

SELECT * FROM mangrove_conservation_records WHERE location LIKE '%Jaya%';

Penjelasan Query:

  • LIKE - Operator untuk pencocokan pola teks
  • % - Wildcard yang mewakili nol atau lebih karakter
  • Mencari lokasi yang mengandung kata "Jaya"

Output Contoh:

conservation_id location area_ha
C001 Aceh Jaya 50
C010 Kubu Raya 70

Analisis: Berguna untuk pengelompokan proyek berdasarkan pola nama lokasi, terutama untuk analisis regional dalam sistem blockchain.

TIPS BAB 5

Indeks untuk Percepatan Filter

Buat indeks pada kolom yang sering difilter (area_ha, carbon_credits) untuk meningkatkan performa query.

Hindari Fungsi pada Kolom WHERE

Penulisan: WHERE UPPER(location) = 'ACEH JAYA' akan menghindari penggunaan indeks. Lebih baik: WHERE location = 'Aceh Jaya'

Urutan Kondisi WHERE

Letakkan kondisi yang paling selektif pertama untuk optimasi performa, contoh: WHERE carbon_credits > 300 AND area_ha > 50
