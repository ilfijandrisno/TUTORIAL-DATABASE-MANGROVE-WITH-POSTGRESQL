Bab 8: Mengagregasi Data: GROUP BY dan Fungsi Agregat

Pada bab ini, kita akan mempelajari cara merangkum data konservasi mangrove menggunakan fungsi agregat seperti COUNT, SUM, dan GROUP BY untuk menghasilkan wawasan statistik, seperti total kredit karbon per lokasi, penting untuk evaluasi kontribusi regional dalam sistem blockchain.

Deskripsi Kasus

Organisasi ingin menghitung total kredit karbon per lokasi untuk mengevaluasi kontribusi regional terhadap pasar karbon blockchain, membantu alokasi pendanaan dan promosi proyek.

Langkah untuk Pemula

1. Hitung Jumlah

SELECT COUNT(*) AS total_records
FROM mangrove_conservation_records;

Penjelasan Query:

  • COUNT(*) - Menghitung total jumlah baris dalam tabel.
  • AS total_records - Memberikan alias untuk hasil perhitungan.
  • FROM mangrove_conservation_records - Menentukan tabel sumber data.

Output Contoh:

total_records
20

Analisis: Query ini memberikan gambaran keseluruhan jumlah proyek aktif, mendukung analisis skala nasional untuk strategi blockchain.

2. Jumlahkan Kredit

SELECT SUM(carbon_credits) AS total_credits
FROM mangrove_conservation_records;

Penjelasan Query:

  • SUM(carbon_credits) - Menjumlahkan nilai kolom carbon_credits.
  • AS total_credits - Memberikan alias untuk hasil penjumlahan.
  • FROM mangrove_conservation_records - Menentukan tabel sumber data.

Output Contoh:

total_credits
5360

Analisis: Query ini menghitung total kredit karbon semua proyek, memberikan potensi pendapatan karbon untuk negosiasi blockchain.

Untuk Pengguna Menengah

Grup berdasarkan Lokasi

SELECT location, SUM(carbon_credits) AS total_credits
FROM mangrove_conservation_records
GROUP BY location;

Penjelasan Query:

  • GROUP BY location - Mengelompokkan data berdasarkan kolom location.
  • SUM(carbon_credits) - Menjumlahkan kredit karbon untuk setiap kelompok lokasi.
  • AS total_credits - Memberikan alias untuk hasil penjumlahan.

Output Contoh:

location total_credits
Aceh Jaya 250
Takalar 370
Labuhan Batu 400

Analisis: Query ini membantu mengidentifikasi wilayah dengan kontribusi karbon tinggi, mendukung alokasi dana blockchain regional.

Untuk Pengguna Tingkat Lanjut

Gunakan HAVING

SELECT location, SUM(carbon_credits) AS total_credits
FROM mangrove_conservation_records
GROUP BY location
HAVING SUM(carbon_credits) > 500;

Penjelasan Query:

  • GROUP BY location - Mengelompokkan data berdasarkan lokasi.
  • HAVING SUM(carbon_credits) > 500 - Memfilter kelompok dengan total kredit karbon di atas 500.
  • SUM(carbon_credits) - Menjumlahkan kredit karbon per lokasi.

Output Contoh:

location total_credits
Labuhan Batu 400

Analisis: Query ini memfokuskan investasi blockchain pada wilayah dengan dampak karbon besar, meningkatkan efisiensi pendanaan.

TIPS BAB 8

Gunakan Fungsi Agregat dengan Tepat

Gunakan fungsi seperti COUNT, SUM, dan AVG untuk merangkum data secara efisien, sesuai kebutuhan analisis.

Sertakan Kolom Non-Agregat di GROUP BY

Pastikan semua kolom non-agregat (misalnya, location) ada di klausul GROUP BY untuk menghindari error sintaks.

Optimasi dengan Indeks

Buat indeks pada kolom yang sering digunakan di GROUP BY atau HAVING untuk meningkatkan performa query.
