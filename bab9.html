Bab 9: Subquery untuk Pemfilteran Lanjutan

Pada bab ini, kita akan mempelajari cara menggunakan subquery untuk pemfilteran kompleks pada data konservasi mangrove, seperti mengidentifikasi proyek dengan kredit karbon di atas rata-rata nasional, penting untuk prioritas pendanaan dalam sistem blockchain.

Deskripsi Kasus

Organisasi ingin mengidentifikasi proyek dengan kredit karbon di atas rata-rata nasional untuk prioritas pendanaan blockchain, memastikan hanya proyek berkinerja tinggi yang dipromosikan.

Langkah untuk Pemula

1. Subquery Sederhana

SELECT conservation_id, location
FROM mangrove_conservation_records
WHERE carbon_credits > (SELECT AVG(carbon_credits) FROM mangrove_conservation_records);

Penjelasan Query:

  • SELECT conservation_id, location - Memilih kolom ID dan lokasi dari tabel utama.
  • WHERE carbon_credits > (SELECT AVG(carbon_credits) ...) - Memfilter baris dengan kredit karbon di atas rata-rata, dihitung oleh subquery.
  • SELECT AVG(carbon_credits) - Subquery menghitung rata-rata kredit karbon.

Output Contoh:

conservation_id location
C006 Labuhan Batu
C014 Sampit
C002 Takalar

Analisis: Query ini menyoroti proyek berkinerja tinggi (misalnya, kredit > rata-rata 268) untuk pendanaan blockchain, memastikan investasi pada proyek dengan dampak besar.

Untuk Pengguna Menengah

Subquery dalam IN

SELECT conservation_id, location
FROM mangrove_conservation_records
WHERE conservation_id IN (
    SELECT conservation_id
    FROM blockchain_transactions
    WHERE carbon_credits_transferred > 300
);

Penjelasan Query:

  • WHERE conservation_id IN (...) - Memfilter proyek berdasarkan daftar conservation_id dari subquery.
  • SELECT conservation_id FROM blockchain_transactions - Subquery mengambil ID proyek dengan transaksi >300 kredit.

Output Contoh:

conservation_id location
C002 Takalar
C006 Labuhan Batu

Analisis: Query ini mengidentifikasi proyek aktif di pasar blockchain dengan transaksi besar, mendukung strategi promosi kredit karbon.

Untuk Pengguna Tingkat Lanjut

Correlated Subquery

SELECT m.conservation_id, m.location
FROM mangrove_conservation_records m
WHERE EXISTS (
    SELECT 1
    FROM conservation_validators v
    WHERE v.conservation_id = m.conservation_id
    AND v.validation_status = 'Approved'
);

Penjelasan Query:

  • WHERE EXISTS (...) - Memeriksa keberadaan baris di subquery yang berkorelasi dengan tabel utama.
  • SELECT 1 FROM conservation_validators v - Subquery mencari proyek tervalidasi dengan status 'Approved'.
  • v.conservation_id = m.conservation_id - Mengkorelasikan subquery dengan tabel utama.

Output Contoh:

conservation_id location
C001 Aceh Jaya
C002 Takalar

Analisis: Query ini memastikan hanya proyek tervalidasi yang digunakan untuk transaksi blockchain, meningkatkan kepercayaan investor.

TIPS BAB 9

Optimasi Performa Subquery

Gunakan subquery dengan hati-hati pada tabel besar; pertimbangkan JOIN sebagai alternatif untuk performa lebih baik.

Gunakan EXISTS untuk Efisiensi

Correlated subquery dengan EXISTS lebih efisien untuk memeriksa keberadaan data dibandingkan IN pada dataset besar.

Indeks pada Kolom Subquery

Buat indeks pada kolom yang digunakan di subquery (misalnya, conservation_id) untuk mempercepat eksekusi.
