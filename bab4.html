Bab 4: Query SELECT Dasar

Bab ini memperkenalkan query SELECT dasar untuk mengambil data dari tabel konservasi mangrove, penting untuk analisis awal sebelum memproses data lebih lanjut dalam sistem blockchain.

Deskripsi Kasus

Organisasi ingin melihat data konservasi mangrove untuk laporan awal, seperti lokasi dan kredit karbon, untuk mengevaluasi proyek yang layak untuk transaksi blockchain.

Langkah untuk Pemula

1. SELECT Sederhana

SELECT conservation_id, location, carbon_credits FROM mangrove_conservation_records;

Output Contoh:

conservation_idlocationcarbon_credits
C001Aceh Jaya250
C002Takalar370

Analisis: Query ini mengambil 3 kolom penting untuk laporan awal, memungkinkan analis melihat distribusi kredit karbon per lokasi.

2. Ambil Semua Kolom

SELECT * FROM mangrove_conservation_records;

Kegunaan: Berguna untuk inspeksi cepat seluruh struktur data sebelum analisis lebih detail.

Untuk Pengguna Menengah

Gunakan Alias Kolom

SELECT conservation_id AS id, location AS situs, carbon_credits AS kredit FROM mangrove_conservation_records;
Untuk Pengguna Tingkat Lanjut

Gunakan DISTINCT

SELECT DISTINCT location FROM mangrove_conservation_records;

TIPS BAB 4

Hindari SELECT * dalam Produksi

Selalu sebutkan kolom eksplisit dalam query produksi untuk efisiensi.

Gunakan LIMIT untuk Data Besar

Tambahkan LIMIT 100 ketika eksplorasi dataset besar.
